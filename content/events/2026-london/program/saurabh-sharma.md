Eight months out of university, RHCE certificate in hand, I joined a 3-person startup and was handed the entire platform. No senior engineer to ask. No runbook. Just me, GitHub Actions docs, and a blank repo.
What followed was two years of making real architectural decisions, i.e. CI/CD pipelines, microservice structure, database patterns, VPC mirroring on AWS, not because I was assigned them, but because nobody else would. I shipped a 30-minute deployment pipeline and thought it was fine. I built a nightly infrastructure teardown to save costs instead of fixing observability. I made it work, and I learned by breaking it.
This talk is for platform engineers who've been the only one in the room. It's about what you learn when execution beats perfection, and how you only understand the gaps later ( when you're somewhere better. )
